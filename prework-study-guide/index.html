Prework Study Guide

HTML

  • Elements:
  • The head element contains information, or metadata, that helps render the page correctly. Elements included in this element are not visible to the user.
  • The meta element contains information about the page that is used by the browser.
  • The body element represents the visible content shown to the user. All elements we want the user to see and interact with must be placed inside this element.
  • The title element defines the title for the webpage, which can be seen in the browser tab. It is required and very important for SEO.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A period can be used to designate a class selector, so that nay HTML element assigned to that class will gain the property assignments from the CSS.

  • Ask about why header and footer are both 120px when example image has footer height at ~40px

Git

  • Git CLI commands:
  • pwd: displays the current working directory
  • ls: displays a list of files in the current working directory
  • cd: navigate between directories
  • cd ..: navigates one level backwards in the directory path
  • mkdir: creates a new directory
  • touch: creates a new file within a directory
  • open or start: opens or starts a file in the directory
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: the add command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes
  • git commit -m "commit message": creates a commit that captures a snapshot of the staged changes

  • Author identity unknown error - run the below two commands:
  • git config --global user.email "myemail@email.com"
  • git config --global user.name "mygitusername"
